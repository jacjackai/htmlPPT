## 部署注意事项

### 缓存

- 确保正确的缓存头
- 使用缓存破坏技术（文件哈希）
- 考虑使用Service Worker缓存

### 安全

- 启用HTTPS
- 设置正确的CSP头
- 避免内联脚本
- 使用安全的Cookie

### 性能

- 启用Gzip压缩
- 优化图片
- 使用CDN
- 启用HTTP/2

### 兼容性

- 支持所有现代浏览器
- 测试移动设备
- 考虑旧浏览器
- 提供回退机制

## 部署指南

### 本地开发

```bash
# 直接打开（零依赖）
open src/index.html

# 使用开发服务器
npm run dev

# 使用Python服务器
python -m http.server 8000 -d src
```

### 生产构建

```bash
# 构建CSS
npm run build:css

# 构建JS
npm run build:js

# 完整构建
npm run build
```

### 部署选项

1. **GitHub Pages**
   ```bash
   npm run deploy
   ```

2. **Netlify/Vercel**
   - 拖放dist/目录
   - 配置自动部署

3. **自托管**
   - 复制dist/到服务器
   - 配置Nginx/Apache
   - 设置HTTPS

## 问题排查

### 常见问题

1. **构建失败**
   - 检查Node.js版本
   - 删除node_modules并重新安装
   - 检查package.json依赖

2. **部署后空白页**
   - 检查控制台错误
   - 验证文件路径
   - 检查缓存问题

3. **功能不工作**
   - 检查浏览器兼容性
   - 验证依赖加载
   - 检查网络请求

4. **性能问题**
   - 检查内存泄漏
   - 优化大型幻灯片
   - 启用生产模式
